# 🧩 一、项目概述（Project Overview）

**项目名称：**
👉《小熊虫回忆 · 贴纸簿》

**产品定位：**
一个基于“Reference Code 解锁”的**互动式回忆收集平台 + 轻社交留言系统**

**核心理念：**

> 用“贴纸收集 + 私密代码”驱动用户探索与情感沉淀

---

# 🎯 二、业务目标（Business Objectives）

## 1. 核心目标

* 提供一种**情绪化 / 游戏化的回忆记录方式**
* 增强用户参与感（收集、解锁、互动）
* 建立“私密但可分享”的内容体系

## 2. 次级目标

* 提高用户停留时长（浏览 + 评论）
* 提供社交互动（留言 + 图片）
* 支持跨设备恢复（Visitor ID机制）

---

# 👤 三、用户画像（User Persona）

## 目标用户群

* 年轻用户（15–35岁）
* 情绪表达型用户（恋爱、友情、纪念）
* 喜欢收集/打卡/仪式感的人群

---

## 用户动机

| 动机   | 说明           |
| ---- | ------------ |
| 情感记录 | 记录回忆         |
| 探索解锁 | 通过 code 获取内容 |
| 社交互动 | 留言、互动        |
| 收藏成就 | 收集完整贴纸       |

---

# 🧠 四、核心用户流程（User Flow）

### 主流程（核心闭环）：

```
进入页面
 → 自动生成 Visitor ID
 → 浏览贴纸（锁定状态）
 → 输入 Reference Code
 → 解锁贴纸
 → 查看回忆（弹窗）
 → 留言 / 上传图片
 → 继续收集
```

---

### 次流程：

#### 1️⃣ 进度恢复

```
输入 Visitor ID → 拉取进度 → 恢复贴纸状态
```

#### 2️⃣ 评论互动

```
打开贴纸 → 输入文字/图片 → 提交 → 展示评论
```

# 🏗 六、非功能需求（Non-functional Requirements）

## 1. 性能

* 首屏加载 < 3s
* API响应 < 500ms

---

## 2. 安全

⚠️ 当前存在风险：

* Supabase anon key 暴露（前端）
* 无权限控制（RLS未说明）

建议：

* 启用 Row Level Security
* 限制 insert/delete 权限

---

## 3. 可扩展性

* 支持新增贴纸（DB驱动）
* 支持多页内容
* 支持多用户并发

---

## 4. 兼容性

* 移动端优先（已做响应式）
* 支持现代浏览器

---

# ⚠️ 七、当前问题 & 风险分析（重点）

## 1. 安全问题（严重）

* Visitor ID 可伪造
* 评论删除仅靠前端判断

👉 建议：

* 后端校验 visitor_id
* 或引入 auth

---

## 2. 数据一致性

* localStorage + 云端可能冲突

---

## 3. 滥用风险

* code 可被暴力破解
* 评论无审核

---

## 4. 可维护性

* JS过长（单文件）
* 无模块化
